# WS-8 — Add / Edit Projects

**Date:** 2026-06-27
**Branch:** feat/daily-driver-v1
**Spec:** docs/superpowers/specs/2026-06-27-atlas-daily-driver-v2-design.md §4 WS-8

## Goal
Give users a way to create a Project inside a Space (today there is none) and to
edit a project's overview/description from the detail view.

## Current state (verified in code)
- `AppState.spaces: [Space]`, each `Space` has nested `[Project]`.
- `Project` carries `spaceName` + `spaceColor` (must mirror the parent Space).
- `AtlasDB.upsertProject(_:)` already exists (AtlasDB.swift:506) — no DB change needed.
- `SidebarView.spaceSection` renders a header button (toggle) + project rows. No "+".
- `ProjectDetailView` shows `project.overview` read-only; gets a `Project` value copy.
- New files in `Atlas/` + `AtlasTests/` are auto-picked-up by xcodegen (dir sources).

## Changes

### 1. AppState (Atlas/Data/AppState.swift) — model + persistence
- `addProject(toSpaceNamed:name:code:isClass:overview:) -> Project?`
  - find space by name; if none, return nil (append nowhere).
  - build `Project` with `spaceName` = space name, `spaceColor` = space.color.
  - append to that space's `projects`; write-through `db?.upsertProject`.
- `updateProjectOverview(projectID:overview:)`
  - locate project across spaces; set `.overview`; write-through `upsertProject`.

### 2. Sidebar (Atlas/Views/Sidebar/SidebarView.swift)
- Add a "+" button in each space header row (and a context menu "Add Project…")
  that sets a `@State` target space name + presents a sheet.
- New `NewProjectSheet` (Atlas/Views/Sidebar/NewProjectSheet.swift): fields for
  name (required), optional code, `isClass` toggle, overview `TextEditor`.
  On Create → `state.addProject(...)`, navigate to the new project, dismiss.

### 3. ProjectDetailView (Atlas/Views/Project/ProjectDetailView.swift)
- Add `@EnvironmentObject var state: AppState` + local edit state.
- Overview section gains an edit affordance: pencil → `TextEditor` + Save/Cancel.
- Save → `state.updateProjectOverview(projectID:overview:)`.
- Show an "Add overview" affordance when overview is empty (so empty projects are editable).

## Tests (AtlasTests/AppStateProjectTests.swift)
- `addProject` appends to the correct space and to no other; nil for unknown space.
- created project's `spaceName`/`spaceColor` mirror the parent space.
- `updateProjectOverview` changes only the target project's overview.
