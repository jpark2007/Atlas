# Atlas iOS v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the full Atlas iOS v1 companion (Capture · Schedule · Tasks · Settings · widgets · local notifications) per `docs/specs/11-mobile-companion.md`, styled Editorial Minimal · light · clay.

**Architecture:** Pure SwiftUI views in the existing `AtlasMobile` target consuming the shared `AtlasCore` package (models, Supabase auth/DB, AI capture, agenda/grouping logic). One `MobileStore` ObservableObject owns session + data snapshot. Widgets read a JSON snapshot from an app group. No new backend.

**Tech Stack:** SwiftUI (iOS 17), AtlasCore (local SwiftPM), XcodeGen, `SFSpeechRecognizer`, `UserNotifications`, WidgetKit.

## Global Constraints

- **Deployment:** iOS 17.0 · Swift 5 · XcodeGen project (`project.yml` → run `xcodegen generate` after editing it; if `Config/Secrets.xcconfig` is missing first run `cp Config/Secrets.example.xcconfig Config/Secrets.xcconfig`).
- **Build check (must pass before every commit):**
  `xcodebuild -project Atlas.xcodeproj -scheme AtlasMobile -configuration Debug -destination 'generic/platform=iOS Simulator' build CODE_SIGNING_ALLOWED=NO -quiet`
  AND the macOS app must stay green:
  `xcodebuild -project Atlas.xcodeproj -scheme Atlas -configuration Debug -destination 'platform=macOS' build CODE_SIGNING_ALLOWED=NO -quiet`
- **Do NOT touch** the Mac app (`Atlas/`) or `AtlasTheme` in `AtlasCore/Sources/AtlasCore/Theme.swift`. Additive AtlasCore changes are allowed only where a task says so.
- **Design system — Editorial Minimal · LIGHT · clay** (from `docs/experiments/ui-style-directions-v2.html`, style 4). Exact tokens:
  - bg `#fbfaf7` · ink `#1a191d` · muted `#6c6a72` · faint `#9a98a0`
  - hairline `black 8%` · strong rule: 1.5 pt solid ink (header underline, outlined controls)
  - accent clay `#d97757`; accent **text** uses `#b04f2f` (AA on light); space colors come from each `Space.color` / `ColorToken` — never hardcoded
  - radii: card 24 · control 19 · chip 13 (continuous corners)
  - type: SF Pro **Rounded** (`.fontDesign(.rounded)`) — screen title 31/heavy tracking −0.03em · row title 15.5/semibold · times 14/bold `.monospacedDigit()` · caps labels 10.5–12/bold +0.08em uppercase · meta 13/medium
  - **RULES:** accent = live/NOW/brand ONLY, never a button fill. Capture affordance = refined glyph, never a solid fill. Controls are transparent with 1.5 pt ink outlines. No card chrome — content sits on the bg, separated by rules.
- **Data correctness (CLAUDE.md §5):** an item's source/read-only status must reflect where it actually came from. Never hardcode source labels.
- **Copy tone:** calm, short. Tab names exactly: `Schedule` · `Capture` · `Tasks` (+ gear, not a tab). App opens on Schedule.
- **UI verification:** a green build does not prove UI. Each UI task ends with booting an iPhone simulator, installing, and screenshotting the screen(s) via `xcrun simctl`. Screenshots go to the session scratchpad, not the repo.
