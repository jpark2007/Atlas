# Doc-tabs v2 Implementation Plan — import/pending/dupes/images/links/tables

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Round out per-tab Google Docs sync: instant pull at import, real "Sync now", pending notes locked, one-Doc-one-note across projects, images re-hosted/displayed/preserved (making image tabs text-editable), pipe-tables rendered as grids, links clickable, and the Settings flag toggle + connection badge.

**Architecture:** Extract google-sync's private pull machinery into `_shared/google_pull.ts` so `drive-import` (inline pull at import) and a new `reference-pull` edge function ("Sync now" with teeth) can reuse it. Migration `0023` re-keys `doc_note_tabs` by `note_id`, adds `doc_note_images` and a private `doc-images` Storage bucket. Images are downloaded at pull time (contentUri is fresh then), stored in Storage, shown in the editor, and re-inserted at write time via `insertInlineImage` with a short-lived signed URL + preserved size — so an uncropped-image tab no longer locks. Duplicates: importing an already-imported Doc attaches the existing note to the new project; baselines update by `drive_file_id` across sibling refs.

**Tech Stack:** Deno/TS edge functions, Postgres migration 0023 + Storage, Swift/SwiftUI (macOS 14, XcodeGen).

### Task 1: Migration 0023 — note-keyed tabs, images table, Storage bucket

**Files:**
- Create: `supabase/migrations/0023_doc_images_and_note_keyed_tabs.sql`

**Interfaces (produces):** `doc_note_tabs.note_id` (backfilled, unique `(note_id, tab_id)`); table `doc_note_images(id, user_id, note_id, tab_id, object_id, storage_path, width_pt, height_pt, crop_locked, created_at)` unique `(note_id, object_id)`, owner-read RLS; Storage bucket `doc-images` (private) + owner-read object policy.

- [ ] **Step 1: Write the migration**

```sql
-- 0023_doc_images_and_note_keyed_tabs.sql
-- v2: (a) key doc tabs by NOTE so one Doc imported into many projects shares one
-- note + one tab set; (b) doc_note_images — re-hosted copies of Doc images
-- (downloaded at pull time while contentUri is fresh, stored in the private
-- doc-images bucket, re-inserted into the Doc at write time); (c) the bucket.

-- (a) note-keyed tabs
alter table doc_note_tabs add column if not exists note_id uuid references notes(id) on delete cascade;
update doc_note_tabs t
   set note_id = r.note_id
  from project_references r
 where r.id = t.reference_id and t.note_id is null;
delete from doc_note_tabs where note_id is null;  -- orphans (reference without note)
alter table doc_note_tabs alter column note_id set not null;
alter table doc_note_tabs alter column reference_id drop not null;
create unique index if not exists doc_note_tabs_note_tab_key on doc_note_tabs (note_id, tab_id);
create index if not exists doc_note_tabs_note_idx on doc_note_tabs (note_id);

-- (b) image map
create table if not exists doc_note_images (
  id            uuid primary key default gen_random_uuid(),
  user_id       uuid not null references auth.users(id) on delete cascade,
  note_id       uuid not null references notes(id) on delete cascade,
  tab_id        text not null,
  object_id     text not null,           -- Docs inlineObject id at pull time
  storage_path  text not null,           -- doc-images object key: <user_id>/<note_id>/<object_id>.<ext>
  width_pt      numeric,                 -- EmbeddedObject.size at pull, preserved on re-insert
  height_pt     numeric,
  crop_locked   boolean not null default false, -- image has crop/rotation/adjustments -> tab read-only
  created_at    timestamptz not null default now(),
  unique (note_id, object_id)
);
create index if not exists doc_note_images_note_idx on doc_note_images (note_id);
alter table doc_note_images enable row level security;
drop policy if exists "doc_note_images: owner reads" on doc_note_images;
create policy "doc_note_images: owner reads" on doc_note_images
  for select using (user_id = auth.uid());

-- (c) private bucket + owner read (writes are service-role only, which bypasses RLS)
insert into storage.buckets (id, name, public)
values ('doc-images', 'doc-images', false)
on conflict (id) do nothing;
drop policy if exists "doc-images: owner reads" on storage.objects;
create policy "doc-images: owner reads" on storage.objects
  for select using (bucket_id = 'doc-images' and (storage.foldername(name))[1] = auth.uid()::text);
```

- [ ] **Step 2: Static check** — `grep -c "create table\|create policy\|create unique index\|insert into storage.buckets" supabase/migrations/0023_doc_images_and_note_keyed_tabs.sql` → expect `5`.
- [ ] **Step 3: Commit** — `feat(db): note-keyed doc tabs + doc_note_images + private doc-images bucket (0023)`

- [ ] **Step 2: Inline pull.** After the reference insert succeeds (after line 337): for each NEWLY-created doc_note reference that has no baseline yet (fresh import, not the dedupe-attach case), best-effort pull:

```ts
  // Inline first pull — the imported note lands populated instead of waiting for
  // the 5-min cron. Best-effort: on any failure the row stays pending and the
  // cron picks it up (same code path, shared module).
  let pulled = 0;
  try {
    const clientId = Deno.env.get("GOOGLE_CLIENT_ID");
    const clientSecret = Deno.env.get("GOOGLE_CLIENT_SECRET");
    if (clientId && clientSecret && newDocNoteRefs.length) {
      const accessToken = await mintAccessToken(admin, userId, clientId, clientSecret);
      const runISO = new Date().toISOString();
      for (const ref of newDocNoteRefs) {
        try {
          await pullDocNoteReference(admin, userId, accessToken, ref, runISO, false);
          pulled++;
        } catch (_e) { /* stays pending; cron retries */ }
      }
    }
  } catch (_e) { /* no connection / vault miss: stays pending */ }
```
Response becomes `{ ok: true, imported, skipped, pulled }`.

### Task 5: Image pipeline — pull-time re-host, classifier relaxation, write-time re-insert

**Interfaces:**

- Pull hook (google_pull): for each image in a writable tab, fetch `contentUri` (no auth header; on 401/403 retry once with Bearer), sniff content-type (png/jpeg/gif; anything else → flag tab read-only `"unsupported image format"`), upload to Storage `doc-images` at `<userId>/<noteId>/<objectId>.<ext>` via service-role client `storage.from("doc-images").upload(path, bytes, { contentType, upsert: true })`, upsert `doc_note_images`. - Renderer change: `renderRequests(tabId, endIndex, markdown, images?: Record<string, {uri, widthPt?, heightPt?}>)`. A line matching `/^!\[image:([^\]]+)\]$/`:

  - known → `{ insertInlineImage: { location: { tabId, index }, uri, objectSize? } }` where objectSize uses `{ magnitude, unit: "PT" }` pairs when dimensions exist; the image occupies **1 UTF-16 unit**; then unless it's the last line, `insertText` `"\n"` at `index+1`; index advances 2 (or 1 if last). Update the drift-regression test accordingly (image lines contribute 1 unit + newline, not their placeholder text).
- Writeback: before rendering, load `doc_note_images` for the note + tab, generate a signed URL per needed image (`createSignedUrl(path, 600)`), build the `images` map. After a successful batchUpdate, RE-PULL the tab's images? NO — YAGNI: object ids change on re-insert; instead let the next cron pull re-harvest ids/URIs (it re-baselines anyway via sibling update; the note's stored bytes don't change). Document this in a comment.
- Editor display is Task 7.
