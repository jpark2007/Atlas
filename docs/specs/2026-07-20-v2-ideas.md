retention/academic-outcomes) into one durable record. This is the decision doc: what's
confirmed, what to build in what order, the full inventory with verdicts, and the dead list

> **⚠️ GRADE-CALC CAVEAT (load-bearing).** Do NOT design the grade calculator around a Canvas
> API pull — that path is confirmed dead at Rutgers AND Princeton (quote above). Build it as a
> **manual-first Grade Ledger**: the student enters score + weight per returned assignment;

> The "final grade calculator" is then just a query on that ledger. Manual entry is not a
> downgrade — it's precisely what makes the grade history live only in Atlas (the retention thesis).

## ii. Recommended portfolio (top 6, sequenced)

The spine is one data structure — the **Grade Ledger** — feeding retention, growth, and the
payment trigger at once. Build in this order:

1. **Syllabus ingest + Canvas cross-check** *(confirmed)* — the universal engine. Hardening the
   parser de-risks the booth, the ledger's weight-seeding, join-code spaces, and reading-debt at
   once. **Build first.**
2. **Grade Ledger → shadow transcript** *(retention #1, ~3–4 wks)* — per-class weighted log →
   live class grade, semester + cumulative GPA across every term. Strongest switching cost on the
   whole list: leaving = abandoning your only unified academic history. Manual-first; weights
   seeded by #1.
3. **Grade-weighted priority queue** *(retention #2, ~2–3 wks)* — deterministic (weight ×
   deadline × effort), NOT AI. The transparent answer to "why is this better than Apple
   Reminders." Depends on #2 for weights.
4. **Weekly receipt + software "Today Card" wallpaper** *(confirmed receipt + growth #3)* — the
   weekly ritual plus a daily ink-on-paper wallpaper/lock-screen render. Turns every laptop in a
   lecture hall into a billboard, and **validates demand for the paper aesthetic before any
   hardware BOM is spent.**
5. **"What I need on the final" share card** *(growth #4, ~3–4 days)* — a cheap bolt-on on the
   ledger. Shares a *target* ("I need a 71% — pray for me"), not an achieved grade — low privacy
   risk, high meme energy.
6. **Parent Progress Portal** *(retention #5, ~3–4 wks)* — the monetization trigger ("the feature
   that makes a student tell their parents to pay"). Reuses existing sharing infra. **Opt-in,
   granular, framed as bragging to the people funding you — never monitoring.** Build only after
   the ledger lands.

Running alongside (tactic, not a build): **Syllabus-Week Setup Booth** — near-free, and it
dogfoods the ingest hardening from #1.

Hardware is gated behind the "get users before hardware" trigger (§iv).

## iv. Hardware line strategy

**Two-product launch thesis:** Dispatch + Standfile ship together. **Dispatch** is the hero — a
palm-sized anodized thermal printer that makes the weekly receipt physical (Sunday week print,
morning day-dispatch, focus-session stamps). **Standfile** is the no-cert gateway that makes the
paper matter and gets a cheap object into hands. Every *other* object adds a separate FCC/CE file
to maintain, so the disciplined line is exactly these two — and Standfile needs **no certification
at all** (no radio, no battery).

**Certification note:** for a 2-person team, every radio must be a **pre-certified module**
(ESP32/Nordic) → ~$500–2k FCC Part 15B *verification* instead of $5–15k intentional-radiator
testing. Dispatch (modular WiFi) and the Commit Dock (modular BLE) each carry ~$1.5k cert;
Standfile/Rail/day-card-stand carry none. *(Cert figures are the hardware lens's estimates —
price-check before committing.)*

**Economics reframe (debate verdict):** do **not** bet the model on branded thermal-paper refill
revenue. Thermal rolls are a commodity (~$0.60); students will buy generic, and a razor-blade
model fails when anyone can substitute the blade. Position Dispatch as a **one-time $149–179 object
with healthy hardware margin**; consumables are a nice-to-have, not the thesis. Also: thermal fades
— Dispatch is a *delight object*, not an archival record. Standfile's cotton-rag refill decks
($9/60) have better margin logic but are still gravy, not the plan.

**Software validates hardware:** ship the confirmed **weekly receipt** and the software **Today
Card wallpaper** first. If the ink-on-paper week-in-review gets shared and loved as pixels, that's
the demand signal that justifies Dispatch's BOM. If it doesn't, we've spent ~2 weeks instead of a
hardware run.

**Maintainer's notes carried forward:** **focus dial** stays listed but low priority ("spinny
timers exist"). **E-ink day card** is liked; plug-in battery is acceptable — treat it as the
hardware expression of the same Today Card the software proves out first. **NFC tags / focus dock /
semester desk mat / BLE button** stay on the seed list as future Rail-family modules.
